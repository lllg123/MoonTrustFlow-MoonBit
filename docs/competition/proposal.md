# MoonTrustFlow 一页项目申报书

## 1. 项目名称

MoonTrustFlow：MoonBit Policy-as-Code 与可信数据流治理工具包

## 2. 项目简介

MoonTrustFlow 是一个面向 MoonBit 生态的轻量策略评估工具包。项目使用
`.mtf` 规则文件描述数据源、危险汇点、可信边界、净化控制点、数据流边、
禁止路径、必经控制点和例外策略，并在 MoonBit 中完成解析、路径评估与
可解释报告输出。

项目不是通用静态分析器，也不是依赖扫描器或 CI 工作流检查器。它的定位是
工程治理基础组件：用可读的策略模型把架构约束和数据流要求沉淀下来，便于
在代码评审、CI 审计、服务边界检查和安全教学中复用。

## 3. 项目方向与适用场景

项目方向属于 MoonBit 生态中的工程工具、策略评估和基础软件补位。适用场景：

- 服务输入、网关、编码器、渲染器、数据库等组件之间的数据流治理；
- CI 中检查关键路径是否经过网关、编码器、脱敏器等控制点；
- 架构评审中表达“禁止直连”“必须经过边界”“允许已审查路径”等规则；
- 后续接入 MoonBit AST、调用图或架构图导出结果，形成更完整的治理链路。

## 4. 拟实现的核心功能

- 解析 `.mtf` 策略模型；
- 支持 `source`、`sink`、`sanitizer`、`boundary`、`node`、`edge`；
- 支持 `deny` 禁止路径、`require through=` 必经控制点、`allow` 精确例外；
- 支持 `severity=high|medium|low` 风险等级；
- 构建有向流图并评估策略命中情况；
- 输出稳定的 CLI/CI 文本报告；
- 提供 MoonBit 测试集、README、设计文档、验收清单和申报材料。

## 5. 原创性说明

本项目为原创项目，不移植已有开源项目。选题避开了常见格式解析器、通用
数据流分析框架、依赖风险扫描器和 CI 工作流检查器等容易重合方向，聚焦
“策略规则 + 可信流模型 + 合规报告”的工程治理场景。规则语法、数据结构、
策略评估算法、报告格式与测试样例均围绕 MoonBit 生态重新设计。
