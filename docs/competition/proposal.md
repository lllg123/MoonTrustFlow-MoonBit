# MoonTrustFlow 一页项目申报书

## 项目名称

MoonTrustFlow：MoonBit 轻量级数据流安全分析与污染传播检测框架

## 项目简介

MoonTrustFlow 是一个面向 MoonBit 生态的软件分析基础库。项目通过小型 `.mtf` 规则语言描述敏感源、危险汇点、净化器、数据流边和允许路径，并在 MoonBit 中实现污染传播分析，输出可解释的 source-to-sink 风险路径。

本项目定位为面向规则模型的数据流安全分析内核，不夸大为完整编译器级静态分析器。它适合作为后续 AI 生成代码审查、可信工具链实验、服务安全建模和轻量级 CI 安全检查的基础组件。

## 项目方向与适用场景

项目贴合 MoonBit 生态中“软件分析框架、工程工具、基础软件生态补位”的方向。MoonBit 生态仍需要更多面向开发者工具链的基础库，尤其是能表达安全规则、分析依赖路径、生成稳定报告的组件。

适用场景包括：

- 服务端输入到渲染、数据库、命令执行等危险操作的数据流建模；
- AI 生成代码或低代码流程的安全审查；
- CI 中对架构级安全约束进行快速检查；
- 教学、竞赛和工具链实验中的数据流分析示例；
- 后续扩展到 MoonBit AST、调用图和真实项目扫描。

## 计划实现的核心功能

- 解析 `.mtf` 模型文本；
- 支持 `source`、`sink`、`sanitizer`、`node`、`edge`、`allow` 声明；
- 构建有向数据流图；
- 从 source 出发传播污染标记；
- 遇到 sanitizer 截断传播；
- 遇到 sink 输出完整风险路径；
- allow 规则按完整路径压制已知安全链路；
- 提供 CLI 演示、测试、README、CI 和项目申报材料。

## 原创性说明

本项目为原创项目，不移植已有开源项目。项目选择“规则模型 + 污染传播分析”作为切入点，避开常见格式解析器、日志处理库和发布工具等容易重合的方向。核心规则语言、数据结构、分析算法、诊断信息、报告格式和测试样例均围绕 MoonBit 生态重新设计。

## 技术路线

首版采用行级 `.mtf` 解析器，将每一行转换为节点、边或策略。分析阶段使用图遍历从敏感源出发传播污染状态，记录访问路径；当路径到达危险汇点时生成 finding；当路径到达净化器时停止当前分支；当路径被 allow 规则精确匹配时不再报告。

项目结构：

- 根包：核心 API 与分析器；
- `cmd/main`：命令行演示；
- `*_test.mbt`：黑盒测试；
- `docs/competition`：申报与提交材料；
- `.github/workflows`：CI 验证配置。

## 预期成果

项目预期交付一个可运行、可测试、可复现的 MoonBit 软件分析基础库，并形成不少于 10 次有效提交的 GitLink 仓库。后续可扩展真实文件读取、规则分组、风险等级策略、调用图导入、MoonBit AST 适配和 HTML 报告。
